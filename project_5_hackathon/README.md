## 📌 О проекте

### Проблема
Пользователи ищут рецепты в сторонних сервисах — Самокат теряет сценарий использования. Текущий раздел с рецептами в формате сторис неудобен: нет категорий, текстового описания, возможности менять порции и добавлять ингредиенты в корзину.

### Бизнес-цель
Рост удержания пользователей через сценарий **«Самокат как источник рецептов»**.

---

## 🔍 Что было сделано

| Этап | Результат |
|:---|:---|
| **Анализ AS IS** | Выявлены проблемы текущей версии: только сторис, статичная подборка, нет категорий, нет текста рецептов, архив не используется |
| **Проектирование TO BE** | Разработан пошаговый сценарий из 12 шагов (от открытия приложения до оплаты заказа) |
| **Прототип** | Создан прототип в Miro с 10 шагами пользовательского сценария |
| **Техническое задание** | Разработано ТЗ с функциональными и нефункциональными требованиями, стадиями разработки и рисками |
| **Бизнес-модель** | Рассчитаны целевые KPI и бизнес-эффекты |

---

## 📊 Ключевые результаты (KPI)

| Метрика | Целевое значение |
|:---|:---|
| Рост MAU раздела «Рецепты» | **+30%** через 3 месяца |
| Рост конверсии «Просмотр → Корзина» | **+5 п.п.** |
| Сокращение времени сбора корзины | **−40%** |
| Рост сохранённых рецептов | **+25%** |
| Повторно опубликованных архивных рецептов | **не менее 50 шт.** |
