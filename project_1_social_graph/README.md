# Картирование социального графа

## 📌 Задача
Проанализировать коммуникации в команде из 100+ сотрудников для выявления скрытых лидеров, «брокеров знаний» и улучшения кросс-функционального взаимодействия.

## 🛠 Инструменты
- **Gephi** — построение и визуализация графа
- **Excel / Power Query** — подготовка и очистка данных
- **Метрики центральности**:
  - *Betweenness centrality* — выявление «брокеров знаний»
  - *Eigenvector centrality* — оценка влиятельности узлов
  - *Модульность* — выделение кластеров

## 🔍 Подход
1. Собраны данные опросов о том, к кому сотрудники обращаются за советом по различным направлениям.
2. Построен социальный граф, где узлы — сотрудники, рёбра — рекомендации.
3. Рассчитаны метрики центральности для выявления ключевых фигур.
4. Выделены сообщества (кластеры) с помощью модульности.

## 📊 Результаты
- Выявлено **5 кластеров** со связанностью от 12% до 26%
- Определено **5 новых «брокеров знаний»** (сотрудники, которые соединяют разные группы)
- Обнаружены руководители на периферии и эксперты в центре — признак децентрализованного сообщества

## 💡 Ценность для бизнеса
Результаты позволили руководству:
- Точечно назначить модераторов для улучшения коммуникаций
- Сократить время на поиск нужного эксперта внутри компании
- Усилить взаимодействие между отделами
